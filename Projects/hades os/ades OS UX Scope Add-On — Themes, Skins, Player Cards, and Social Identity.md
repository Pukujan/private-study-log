# Hermes / Hades OS UX Scope Add-On — Themes, Skins, Player Cards, and Social Identity

## UX Direction Update

Hermes / Hades OS should not only feel like a utility app.

It should feel like a customizable consumer platform where users can:

```txt
talk to Hermes
collect minions
equip minions into slots
connect socials
receive alerts
customize their profile
showcase their identity
buy or unlock themes, skins, and profile cards
```

The product should feel:

```txt
gamery
smooth
friendly
customizable
social
mobile-first
consumer-readable
```

Not:

```txt
enterprise SaaS
developer ops console
generic chatbot dashboard
technical automation builder
```

---

# Theme System

The app should include a visible theme changer.

This should not be hidden in settings only.

Theme changing is part of the product identity.

Example UI language:

```txt
Choose your realm
Pick your skin
Equip theme
Customize your forge
```

Starter themes:

```txt
🔥 Ember Forge
🌌 Arcade Night
🌿 Forest Grove
🧊 Frost Vault
💖 Candy Rift
```

Each theme can change:

```txt
background
card style
glow color
particle effects
button gradients
minion aura
profile card frame
marketplace preview style
```

MVP should include:

```txt
visible theme changer
2–3 selectable themes
theme preview cards
theme applied to UI
locked premium themes
```

MVP should not include:

```txt
real theme purchases
creator theme uploads
skin trading
paid cosmetic economy
```

---

# Skins and Cosmetic Marketplace

The marketplace should eventually support two categories:

```txt
Functional Marketplace
- minions
- commands
- workflows
- automations
- social helpers
- templates

Cosmetic Marketplace
- themes
- skins
- minion skins
- profile cards
- badges
- profile frames
- UI effects
- animated backgrounds
```

Cosmetics should be optional and should not block basic product use.

They should make the app feel more personal and collectible.

---

# Player Card / Profile Card

Users should have a customizable player card similar in spirit to a Discord profile card.

The profile card should be visual, shareable, and eventually interactive.

Player card can show:

```txt
username
avatar
status
equipped theme
favorite minion
active minions
global commands
badges
creator rank
connected socials
showcase minions
marketplace creations
```

Example:

```txt
Pujan
Forge Builder · Level 7

Equipped Theme:
Ember Forge

Favorite Minion:
Deal Watcher

Showcase:
!sendcatmeme
GitHub Packet Minion
Discord Recap

Badges:
Early Founder
Minion Tamer
Forge Creator
```

MVP should include:

```txt
profile page shell
player card preview
equipped theme display
favorite minion slot
locked badge slots
locked showcase slots
```

MVP should not include:

```txt
public social network
real profile sharing
paid card trading
creator profile economy
```

# Social Network Layer

Hermes / Hades OS can eventually include a lightweight internal social layer.

This does not mean building a full social media platform in MVP.

Future social layer can include:

```txt
public profiles
creator pages
favorite creators
shareable minions
minion showcases
profile cards
creator storefronts
user collections
featured builders
```

The purpose is to support:

```txt
discovering creators
buying minions from favorite developers
showcasing your setup
sharing useful helpers
building trust in marketplace items
```

# Inventory and Loadout UX

The profile and marketplace should connect to the minion inventory system.

Use language like:

```txt
Inventory
Loadout
Equipped
Slots
Global Slots
Social Slots
Owned
Rented
Trial
Locked
```

Users should understand:

```txt
which minions they own
which minions are active
which minions are equipped globally
which minions are attached to Discord / Telegram / GitHub / Email
how many slots are available
what requires upgrade
```

This keeps the product gamery but understandable.

---

# Paywall UX

Paywalls should be natural and tied to inventory expansion or premium cosmetics.

Paywall moments:

```txt
activating more minions than available slots
equipping extra global commands
buying premium minions
renting a minion
buying themes
buying profile card skins
unlocking creator packs
```

Subscription can include:

```txt
extra global minion slots
extra social minion slots
monthly credits
premium theme access
featured minion trials
creator marketplace discounts
```

The paywall should feel like:

```txt
upgrade your loadout
unlock more slots
equip more helpers
customize your profile
```

Not:

```txt
pay to use the app at all
```

---

# MVP UX Boundary

MVP should show the identity of this system but not implement the full economy.

MVP can include:

```txt
mobile-first Hermes home
offline chat
starter minion cards
active minion inventory preview
theme changer
profile/player card preview
locked marketplace preview
locked premium skins
locked social cards
locked creator cards
inbox preview
GitHub task packet minion
manual automation minion
```

MVP should not include:

```txt
real payments
real marketplace transactions
real credits
real creator payouts
real skin purchases
real public social network
real minion rentals
real resale/exchange
real Discord/Telegram bot deployment
```

# Final UX Memory

Hermes / Hades OS should feel like:

```txt
a mobile-first consumer automation companion
where users collect minions,
equip them into social/global slots,
customize their theme and profile,
receive useful alerts,
and eventually buy/share minions and skins through a marketplace.
```

The MVP should make this direction visible and exciting, but only implement the safe core:

```txt
offline chat
starter minions
manual tools
manual automations
theme changer
profile card preview
locked marketplace
locked socials
locked cosmetics
```
