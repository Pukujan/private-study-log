0. One-line model

Cortex  = local memory
        + fetch of REGISTERED URLs only
        + cite / sufficiency gates

Driver  = web discovery + tools + mutation

Bare llm_complete = judge over a pack you already made
                  ≠ research

fanout (this repo) = multi-model BUILD
                   ≠ web-research fanout

1. Target path (contract)

Steps (always readable)

  1. 1 User request
  2. 2 Brain recall (docs / accepted / KEDB)
  3. 3 Tenant / local recall (project, closeouts)
  4. 4 Coverage + freshness + risk decision
    • sufficient → freeze research brief
    • insufficient → driver web discovery
  5. 5 register_source (admin + SSRF checks)
  6. 6 bounded_fetch → local snapshot + SHA-256
  7. 7 Cited local report (gaps visible)
  8. 8 Freeze research brief → freeze contracts
  9. 9 Execute → hidden eval / human → KEDB
flowchart TB
  U["1 · User request"]
  U --> B["2 · Brain recall"]
  B --> L["3 · Tenant / local recall"]
  L --> D{"4 · Coverage + risk?"}
  D -->|sufficient| FR["8a · Freeze research brief"]
  D -->|insufficient| W["5 · Driver web discovery"]
  W --> R["6 · register_source"]
  R --> F["7 · fetch + SHA-256"]
  F --> RP["Cited local report"]
  RP --> FR
  FR --> FC["8b · Freeze contracts"]
  FC --> X["9 · Execute"]
  X --> E["Hidden eval / human"]
  E --> K["KEDB / promotion"]

2. What cortex_research actually does

Steps (always readable)

  1. 1 Question in
  2. 2 Optional Haiku frame → sub-questions
  3. 3 Load sources.yaml registry
  4. 4 select_sources by topics
  5. 5 Fetch only registered URLs (or skip network)
  6. 6 gather_evidence on local index
  7. 7 cite_check → coverage / unanswered
  8. 8 needs_sources?
  9. 9 Optional Haiku summarize
  10. 10 write_report + rebuild tenant index

No step opens the open web by itself. Web only after driver discovers URL → register → fetch.

flowchart TB
  Q["1 · question"]
  Q --> FR{"2 · do_frame?"}
  FR -->|yes| H1["Haiku → sub-questions"]
  FR -->|no| SQ["sub_questions = question"]
  H1 --> REG["3 · load sources.yaml"]
  SQ --> REG
  REG --> SEL["4 · select_sources"]
  SEL --> BF{"5 · do_fetch?"}
  BF -->|yes| GET["fetch REGISTERED urls only"]
  BF -->|no| EV["6 · gather_evidence LOCAL"]
  GET --> EV
  EV --> CC["7 · cite_check"]
  CC --> GAP["8 · needs_sources?"]
  GAP --> SUM{"9 · summarize?"}
  SUM -->|yes| H2["Haiku summarize"]
  SUM -->|no| WR["10 · write_report"]
  H2 --> WR

3. Driver + Cortex + sufficiency

Steps (always readable)

  1. driver User asks (has web / shell tools)
  2. cortex search / scope_pack
  3. cortex cortex_research
  4. branch needs_sources?
    • yes → driver web → register_source → re-run research
    • no → sufficiency path
  5. suff Policy floor → independent review → human (high-risk)
  6. out SUFFICIENT / UNRESOLVED / ABSTAIN
flowchart TB
  U["User question"]
  U --> CS["search / scope_pack"]
  CS --> CR["cortex_research"]
  CR --> NS{"needs_sources?"}
  NS -->|yes| W["Driver web tools"]
  W --> REG["register_source"]
  REG --> CR
  NS -->|no| POL["Policy floor"]
  POL --> IND["Independent review"]
  IND --> HUM["Human high-risk"]
  HUM --> REC["SUFFICIENT / UNRESOLVED / ABSTAIN"]

4. Bare panel vs real research

A — Bare llm_complete (NOT research)

  1. Orchestrator packs corpus / metrics into text
  2. Sol / Grok completion with no tools
  3. Answer only over that pack
flowchart TB
  O1["Orchestrator packs corpus"]
  O1 --> M1["Sol / Grok — no tools"]
  M1 --> A1["Answer over pack only"]

B — Real Cortex research

  1. Tool-capable driver
  2. Local search
  3. needs_sources?
  4. Web discovery
  5. register + fetch + re-run
  6. Sufficiency receipt
flowchart TB
  O2["Tool-capable driver"]
  O2 --> L2["Local search"]
  L2 --> G2["needs_sources?"]
  G2 --> W2["Web discovery"]
  W2 --> R2["register + fetch + re-run"]
  R2 --> S2["Sufficiency receipt"]
