## Bugs Found and Fixed Before Running Anything

The approach was to read all relevant files in parallel first, find every problem, then fix them before touching the backend. Three bugs were found.

### Bug 1: Orphaned function declaration in neo4j.adapter.js

`countGraphByJurisdiction` had a function body with no declaration. The function was called by `seed.js` (to return counts in the seed response) but did not exist as an exported function — just a floating block of code.

```javascript
// Before — orphaned body, would have thrown ReferenceError at call time
  const result = await session.run(
    `MATCH (n) WHERE n.jurisdiction_id = $jurisdictionId ...`
  )
  // ...

// After — proper export
export async function countGraphByJurisdiction(jurisdictionId) {
  const session = driver.session();
  const result = await session.run(
    `MATCH (n) WHERE n.jurisdiction_id = $jurisdictionId ...`
  )
  // ...
}
```

### Bug 2: Wrong Cypher relationship in getRequiredEvidenceTypes

`getRequiredEvidenceTypes` was trying to traverse from Framework to Obligation using `[:CONTAINS]` — a relationship that does not exist in the schema. The correct relationship is `[:HAS_OBLIGATION]`.

```cypher
-- Before (wrong — CONTAINS does not exist in the graph)
MATCH (f:Framework {id: $frameworkId})-[:CONTAINS]->(o:Obligation)

-- After (correct — matches what was seeded in Phase 1)
MATCH (f:Framework {id: $frameworkId})-[:HAS_OBLIGATION]->(o:Obligation)
```

This would have caused evidence coverage to always return 0, making the confidence score systematically lower than it should be and `caps_applied` to incorrectly show the evidence cap.

### Bug 3: Missing jurisdictionId argument in analyze.js

`getGraphContextForChunks` was being called with only `citations` (the chunk IDs) but the function signature requires `(chunkIds, jurisdictionId)`. Without the jurisdiction filter, the graph query would either fail or return context from the wrong jurisdiction.

```javascript
// Before
const graphContext = await getGraphContextForChunks(citations);

// After
const graphContext = await getGraphContextForChunks(citations, jurisdictionId);
```

The UNKNOWN_001 response already has `dynamic_candidates` populated. Phase 3 takes it from there:

```
1. Human calls POST /ontology/expand with approved edges
2. ontology-expander.js runs Cypher MERGE to create Topic node

   - SEMANTICALLY_RELATED_TO edge to nearest known topic
3. Re-run analyze on UNKNOWN_001
4. This time confidence should be >= 0.55 (term now resolves)
5. No longer hits the wall — system learned from the encounter
```

The Cypher that will run:

```cypher
MERGE (t:Topic { id: 'biodiversity_impact' })
ON CREATE SET
  t.name = 'Biodiversity Impact',
  t.status = 'active',
  t.origin = 'dynamic',
  t.jurisdiction_id = 'EU_ESRS_E1'
WITH t
MATCH (existing:Topic { id: 'climate_risk_disclosure' })
MERGE (t)-[:SEMANTICALLY_RELATED_TO { source: 'vector_co_occurrence' }]->(existing);
```

After this runs, the ontology has grown. No redeployment. No schema migration. The graph learned from a human decision.

## Things Worth Remembering From This Session

**Read all files before touching anything.** All three bugs were found by reading first, not by running and seeing what broke. This saved at least an hour of debugging.

**Import smoke test before starting the server.** Four lines of node that verify every export is resolvable. Costs 10 seconds. Saves the confusion of a server that starts but silently fails on the first request because an export is missing.

**The UNKNOWN_001 confidence score came back as 0, not 0.40.** The cap floor is 0.40 but multiple caps triggered simultaneously — `no_citations` (cap 0.45) and `unknown_with_candidates` (cap 0.65) — and the lowest cap rule brought it to 0.45. But with zero evidence coverage the raw score was also 0, so the final was min(0, 0.45) = 0. This is correct behavior. The system had genuinely no evidence to work from.

**Wrong Cypher relationships are silent failures.** `[:CONTAINS]` did not throw an error — it just returned zero results because no such edges exist. The evidence coverage would have been silently 0 on every query until someone noticed the confidence scores were always artificially low. Reading the schema before writing queries is not optional.
